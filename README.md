# 亲情陪伴系统 Family Accompany System

面向老年人的 AI 亲情陪伴项目。项目希望把家庭记忆、老人画像、家人档案、AI 角色扮演、情绪识别和安全边界结合起来，让 AI 回复更像熟悉的家人，而不是泛泛的聊天机器人。

## 项目目标

很多老人真正需要的不是更复杂的工具，而是稳定、熟悉、低压力的陪伴。本项目探索一种“家庭共同维护记忆，AI 按家人语气陪伴老人”的产品形态：

- 帮助老人获得更自然的情感回应和日常陪伴。
- 帮助家人沉淀家庭记忆、生活习惯和沟通方式。
- 在医疗、金钱、死亡等敏感话题上保留明确安全边界。
- 逐步从本地原型演进为可多人协作维护的家庭陪伴产品。

## 期望功能

第一阶段产品化目标参考 `docs/superpowers/specs/2026-05-22-productization-plan.md`：

- 家庭空间：家人可以登录并加入同一个家庭空间，共同维护陪伴资料。
- 成员权限：支持 `owner`、`editor`、`viewer` 三类角色，用于邀请成员、编辑资料和只读查看。
- 老人画像：记录老人的性格、偏好、习惯、健康备注、人生经历和重要记忆。
- 家人档案：记录家人与老人的关系、称呼、性格、说话方式、偏好和家庭关系。
- 家庭记忆：沉淀节日、旅行、日常趣事、重要日期等可用于陪伴对话的记忆素材。
- AI 角色：让 AI 按指定家人的称呼、语气、性格和陪伴方式进行回复。
- 智能导入：家人用自然语言描述家庭信息，系统解析为可编辑的画像、档案和记忆草稿。
- 老人端聊天：老人以简洁界面进行文字聊天，系统结合画像和记忆生成共情回复。
- 语音陪伴：家人上传或录制声音样本，生成克隆声音；聊天回复可播放语音版本。
- 安全校验：对医疗建议、虚假承诺、敏感话题、网络用语和不适老表达进行约束。
- 云端同步：使用 Supabase Auth、Postgres、Storage 和 RLS 隔离家庭数据。

## 当前仓库形态

当前仓库处于“原型验证 + 产品化拆分”阶段：

- `app.py` 保留 Streamlit 单页原型，用于验证对话流程、画像管理、记忆管理和调试信息。
- `engine/` 承载核心陪伴逻辑，包括画像、记忆、评分、策略选择、适老与安全校验。
- `llm/` 封装 DeepSeek 的 OpenAI 兼容接口、Prompt 和自然语言解析。
- `api/` 提供 FastAPI 服务雏形，用于把 Python 对话能力拆成后端接口。
- `web/` 是 Next.js + TypeScript 的正式 Web 应用雏形。
- `supabase/` 保存产品化数据表、权限和云端存储相关迁移。
- `productization/` 保存家庭空间、权限、语音能力和服务层抽象。
- `tests/` 使用 pytest 约束核心行为、API 集成和产品化基础结构。

Streamlit 原型会作为内部调试工具保留；正式用户端规划为 Next.js Web 应用。

## 核心对话流程

1. 老人输入一句话。
2. LLM 识别意图、情绪、对话对象和提及人物。
3. 系统根据当前 AI 角色、家人档案和家庭记忆检索候选上下文。
4. 记忆评分引擎按相关性、共情度、亲密度、安全性和敏感惩罚排序。
5. 策略模块选择安慰、陪伴、转移注意力、鼓励等回复策略。
6. LLM 根据画像、记忆和策略生成回复。
7. 适老与安全模块检查句子长度、称呼、禁忌词、医疗边界和虚假承诺。
8. 前端展示文字回复，产品化版本可进一步播放克隆语音。

## 技术栈

- Python + Streamlit：原型界面和流程编排。
- FastAPI + Pydantic：后端接口层。
- SQLite：当前本地原型持久化。
- DeepSeek OpenAI Compatible API：意图识别、结构化解析和回复生成。
- Next.js + TypeScript：正式 Web 应用方向。
- Supabase：计划中的云端 Auth、Postgres、Storage 和 RLS 权限。
- pytest：单元测试、集成测试和源码约束测试。

## 本地运行

安装 Python 依赖：

```bash
python -m pip install -r requirements.txt
```

配置 DeepSeek API Key。不要把真实 Key 写入代码或文档。

PowerShell：

```powershell
$env:DEEPSEEK_API_KEY="your-key"
python -m streamlit run app.py --server.port 8501
```

运行测试：

```bash
python -m pytest tests/ -v
```

## 项目状态

这是一个持续演进中的 AI 陪伴系统原型。当前重点不是做通用聊天，而是验证“家庭记忆驱动、家人角色扮演、适老表达和安全边界”能否组合成更可信的亲情陪伴体验。
