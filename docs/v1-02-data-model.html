02 — 数据模型设计

四层画像体系：老人 · AI角色 · 家人 · 记忆

1. 老人画像 ElderProfile

单例存储，性别自动推导称呼（女→"妈"，男→"爸"），每轮对话注入回复上下文。

@dataclass class ElderProfile: full_name: str # "宋桂兰" gender: str # "男" | "女" personality: list[str] # ["内向","爱操心"] preferences: list[str] # ["做面食","养花"] habits: list[str] # ["早起","饭后散步"] health_notes: list[str] # ["高血压","忌油腻"] speech_traits: list[str] # ["语速慢","话不多"] life_experiences: list[str] # ["退休工人","带大两个孙子"] important_memories: list[str] # ["儿子考上大学摆了三天酒"] notes: str # 补充

2. AI 扮演角色 PersonaProfile

多角色支持，role_label 主键。模糊名匹配合并，对话时自动切换。

role_label: str # "儿子刘承文"（主键） relation: str # 子女/儿媳/女婿/配偶/孙辈/朋友/护工 appellation: str # "妈"（老人性别推导默认值） personality: list[str] # 12项可选标签 speech_style: list[str] # ["喜欢叫妈","爱问吃了没"] comfort_style: list[str] # 10项陪伴方式

3. 家人档案 FamilyProfile

每个家人独立档案，含与其他家人的关系网。

name: str # "刘承文"（主键） relation: str # 与老人关系 personality: list[str] # 性格 preferences: list[str] # 喜好 habits: list[str] # 习惯 relations: list[dict] # [{"person":"张怀粉","relation":"妻子"}] notes: str # 备注

4. 家庭记忆 MemoryUnit

主语标注，按人物检索。

content: str # 记忆正文 subject: str # "刘承文"——这条记忆是关于谁的 memory_type: str # 事件/习惯/偏好/重要日期/趣事 family_members: list[str] # ["刘承文(子女)","张怀粉(儿媳)"] emotion_tags: list[str] # 8种情感 topic_tags: list[str] # 8种话题 intimacy_weight: float # 0~1 access_count: int # 调用计数
