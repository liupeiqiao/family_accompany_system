# 档案与记忆智能导入设计

## 背景

老人端对话已经可以调用后端 `/api/chat`。下一步需要把 Streamlit 原型里的“粘贴家庭资料、智能解析、预览编辑、一键导入”能力产品化到 Web 前端，让家人可以先维护老人画像、家人档案、AI 扮演角色和家庭记忆，再让老人端对话使用这些资料。

## 目标

- 在 `档案与记忆` 页面提供一个集中式智能导入入口。
- 用户粘贴自然语言家庭资料后，前端调用 `POST /api/parse` 得到结构化结果。
- 前端展示可编辑预览，覆盖老人画像、AI 扮演角色、家人档案、家庭记忆四类数据。
- 用户确认后调用新增的 `POST /api/import`，把编辑后的数据写入现有 SQLite。
- 保存成功后，老人端对话应能通过现有聊天流程读取新导入的画像和记忆。

## 非目标

- 不做账号体系、家庭空间权限、多人协作和云端数据库。
- 不做批量文件上传、OCR、语音导入。
- 不重构 Streamlit 原型的大块逻辑。
- 不把真实 API Key 写入前端、文档、日志或代码。

## 推荐方案

采用“集中式智能导入页 + 后端保存接口”的方案。

`web/src/app/records/page.tsx` 从说明页改成客户端页面，包含输入区、解析按钮、预览编辑区、保存按钮和错误提示。解析继续使用现有 `/api/parse`。保存新增 `/api/import`，请求体直接接收前端编辑后的结构化数据，并在后端复用 `engine/db.py` 的保存能力。

这个方案优先复用现有解析、数据库和聊天链路，避免在第一版引入新的状态系统或云服务。

## 页面行为

1. 用户进入 `档案与记忆` 页面。
2. 在文本框粘贴自然语言资料。
3. 点击“智能解析”。
4. 页面调用 `sendParse`，展示解析结果。
5. 用户可以编辑：
   - 老人画像的基本文本字段。
   - AI 扮演角色的角色名、关系、称呼、性格、说话风格、陪伴方式。
   - 家人档案列表，支持编辑和删除单条。
   - 家庭记忆列表，支持编辑和删除单条。
6. 点击“一键保存”。
7. 页面调用 `importParsedData`。
8. 保存成功后展示成功状态，并提示可进入老人端验证对话效果。

## 后端接口

### `POST /api/import`

请求体字段：

- `family_id`: 默认 `local`。
- `persona`: 可选对象。
- `elder_profile`: 可选对象。
- `family_profiles`: 对象数组，默认空数组。
- `memories`: 对象数组，默认空数组。

响应体字段：

- `ok`: 布尔值。
- `imported`: 对象，包含保存数量，例如 `persona`、`elder_profile`、`family_profiles`、`memories`。

保存规则：

- 空对象或空列表不报错，按未导入处理。
- 字段缺失时使用现有模型默认值或空字符串。
- 家人档案和角色去重优先沿用现有引擎能力。
- 数据库写入失败时返回 API 错误，前端保留当前预览数据。

## 前端接口封装

在 `web/src/lib/backend-api.ts` 中补齐：

- `parseProfileText(request)`
- `importParsedData(request)`

函数应复用现有后端 base URL 逻辑。失败时抛出可读错误，由页面转成中文提示。

## 错误处理

- 解析失败：显示“暂时无法解析，请稍后重试”，不清空输入文本。
- 保存失败：显示“保存失败，请稍后重试”，不清空预览。
- 后端未启动或地址错误：显示“连不上后端服务，请确认 API 服务已启动”。
- 解析结果为空：显示空状态，允许用户修改原文后再次解析。

## 测试

后端测试：

- `/api/import` 能把 persona、elder_profile、family_profiles、memories 写入测试数据库。
- 空请求体不会崩溃，返回导入数量为 0。
- 保存失败时返回明确错误。

前端静态/类型测试：

- `records` 页面调用 `/api/parse` 和 `/api/import` 封装。
- 页面包含解析、编辑预览、保存状态和错误提示。
- `npm run typecheck` 通过。

回归测试：

- 现有 pytest 全量通过。
- 老人端 `sendChat` 调用不受影响。

## 验收标准

- 家人在 Web 页面粘贴一段家庭资料，可以解析出四类结构化数据。
- 解析后的数据可以在页面上编辑和删除单条列表项。
- 点击“一键保存”后数据写入 SQLite。
- 重启或刷新前端后，老人端聊天可以使用新导入的记忆和画像。
- 不需要购买云服务器即可在本地完成初版闭环。
