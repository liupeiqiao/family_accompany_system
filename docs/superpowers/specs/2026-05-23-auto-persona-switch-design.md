# 老人端自动角色切换设计

## 背景

老人端对话已经可以读取多个 AI 角色、老人画像、家人档案和家庭记忆。当前 `generate_chat_reply()` 会根据意图识别结果中的 `talk_to` 匹配 `role_label`，例如 `talk_to="儿子小明"` 时切换到“儿子小明”。这是一版可用雏形，但匹配范围偏窄。

## 目标

当存在多个 AI 角色时，系统应能根据老人本轮话语自动选择最合适的家人角色。比如老人说“小红，我想你了”时，本轮应切换到“女儿小红”；老人说“我想和老伴说会儿话”时，本轮应切换到配偶/老伴角色。

本阶段只做单轮切换，不持久化会话角色状态。后续接入云存储和会话表后，再支持“持续以某个角色对话，直到老人再次切换”。

## 匹配策略

新增一个内部角色匹配器，综合使用：

- `talk_to`：LLM 识别出的对话对象。
- `user_input`：老人原话。
- `mentioned`：LLM 识别出的提及人物。
- `persona.role_label`：如“儿子小明”“女儿小红”。
- `persona.relation`：如“儿子”“女儿”“老伴”“配偶”。
- 家人档案姓名和关系：如“小红 / 女儿”。

匹配优先级：

1. 明确角色名或姓名命中：`小红，我想你了` -> “女儿小红”。
2. 明确关系称呼命中：`我想和老伴说会儿话` -> 配偶/老伴角色。
3. `talk_to` 命中角色名、姓名或关系。
4. 仅提到别人但不是对他说话时，不切换。
5. 多个角色同时命中且无法区分时，保留默认角色。

## Debug 输出

聊天 debug 增加：

- `selected_persona`：本轮最终使用的角色名。
- `persona_switch_reason`：切换原因，如 `direct_name`、`relation_request`、`talk_to`、`default`、`ambiguous`。
- `persona_switch_confidence`：简化置信度，范围 `0.0` 到 `1.0`。

## 测试策略

- 多角色存在时，老人直接呼叫“小红”，应切换到“女儿小红”。
- 老人说“我想和老伴说会儿话”，应切换到配偶/老伴角色。
- 老人只是提到“小红最近忙吗”，但 `talk_to` 仍是陪伴者时，不应误切到小红。
- 保留现有聊天、导入和 prompt 规则测试。
