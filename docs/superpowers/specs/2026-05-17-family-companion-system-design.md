# 亲情陪伴系统 — 设计规格说明

2026-05-17 | 基于专利 `一种亲情记忆驱动的适老共情对话方法` 的核心原型实现

## 范围

核心算法原型 + Streamlit 简易 UI + DeepSeek V4 Pro API 验证。不包含持久化存储，数据通过侧边栏手动输入，内存中管理。不包含多模态输入（仅文本）。

## 技术栈

- **语言**: Python 3.10+
- **UI**: Streamlit（chat + 侧边栏表单）
- **LLM**: DeepSeek V4 Pro（OpenAI 兼容 SDK）
- **依赖**: `streamlit>=1.28`, `openai>=1.0` — 仅两个

## 架构

```
亲情陪伴系统/
├── app.py              # Streamlit 主入口 + 对话循环
├── engine/
│   ├── memory.py       # 家庭记忆库（内存数据结构）
│   ├── persona.py      # 人物画像（静态+动态属性）
│   ├── scorer.py       # 评分引擎 αR+βE+γC+δS-εM
│   ├── strategy.py     # 共情策略选择器（意图×情绪→策略矩阵）
│   └── adaptation.py   # 老年适配规则检查
├── llm/
│   ├── client.py       # DeepSeek API 客户端封装
│   └── prompts.py      # 所有 Prompt 模板集中管理
└── requirements.txt
```

**分层原则**: LLM 负责理解与生成（意图/情绪识别、语义打分补丁、回复生成），Python 负责评分与策略（五因子公式、策略矩阵查表、老年适配规则）。

## 数据模型

### MemoryUnit（家庭记忆单元）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | 唯一标识 |
| content | str | 记忆正文 |
| memory_type | str | 事件/习惯/偏好/重要日期/趣事 |
| family_members | list[str] | 涉及家人 |
| timestamp | datetime\|None | 事件时间（可模糊） |
| emotion_tags | list[str] | 温馨/快乐/感动/搞笑/难忘/遗憾/伤感/兴奋 |
| topic_tags | list[str] | 饮食/旅行/节日/成长/健康/宠物/工作/日常 |
| intimacy_weight | float | 亲密度 0.0~1.0 |
| access_count | int | 累计调取次数 |
| last_accessed | datetime\|None | 最后调取时间 |

### PersonaProfile（人物画像）

静态属性：role_label, relation, appellation, personality (list), speech_style (list), comfort_style (list)

comfort_style 选项：唠家常/撒娇/讲趣事/一起回忆/逗开心/讲道理/转移话题/鼓励/附和倾听/默默陪伴

动态属性：mood_preference (dict), topic_affinity (dict), sensitivity_map (dict)

### ScoreResult（评分结果）

score_r (相关性), score_e (共情度), score_c (亲密度), score_s (安全性), penalty_m (罚分), total

### DialogueContext（运行时上下文）

user_input, intent, emotion, persona, memories, history, selected_memory, strategy

## 核心算法

### 评分公式：Score = αR + βE + γC + δS − εM

默认权重：α=0.30, β=0.25, γ=0.20, δ=0.15, ε=0.10

- **R（相关性）**: 话题标签匹配(40%) + 家人提及匹配(25%) + 时间衰减(15%) + LLM语义打分补丁(20%，仅分差接近时调用)
- **E（共情度）**: 情绪-标签匹配(40%) + 历史策略有效性(35%) + 情感冲突矩阵检查(25%)
- **C（亲密度）**: 关系基础权重(50%) + 记忆亲密度(30%) + 访问热度(20%)
- **S（安全性）**: 风险关键词检测(50%) + 敏感话题匹配(30%) + 医学边界(20%)
- **M（罚分）**: 已故亲人触发(0.5) / 创伤事件触发(0.4) / 近期负面事件(0.3)

### 情感冲突矩阵（E 子项核心）

老人当前情绪 × 记忆情感标签 → 三级判定。E = (兼容记忆数 / 候选总数) × 该子项权重。

| 老人情绪 ↘ 标签 → | 搞笑 | 温馨 | 感动 | 遗憾 | 伤感 | 兴奋 | 日常 | 难忘 |
|-------------------|:----:|:----:|:----:|:----:|:----:|:----:|:----:|:----:|
| **难过**         |  ❌  |  ✅  |  ⚠️  |  ❌  |  ❌  |  ⚠️  |  ✅  |  ✅  |
| **焦虑**         |  ✅  |  ✅  |  ✅  |  ❌  |  ❌  |  ⚠️  |  ✅  |  ⚠️  |
| **孤独**         |  ✅  |  ✅  |  ✅  |  ⚠️  |  ❌  |  ✅  |  ✅  |  ✅  |
| **委屈**         |  ❌  |  ✅  |  ✅  |  ⚠️  |  ⚠️  |  ⚠️  |  ✅  |  ⚠️  |
| **抱怨**         |  ❌  |  ⚠️  |  ✅  |  ⚠️  |  ⚠️  |  ⚠️  |  ✅  |  ⚠️  |
| **思念**         |  ⚠️  |  ✅  |  ✅  |  ❌  |  ❌  |  ✅  |  ✅  |  ✅  |
| **怀疑**         |  ❌  |  ⚠️  |  ⚠️  |  ⚠️  |  ⚠️  |  ⚠️  |  ✅  |  ⚠️  |
| **开心/平静/兴奋**|  ✅  |  ✅  |  ✅  |  ✅  |  ✅  |  ✅  |  ✅  |  ✅  |

**三级判定含义:**

- ✅ **兼容** — E 分不变，正常参与评分
- ⚠️ **降权** — E 分 × 0.4，仅当无 ✅ 记忆时可作为备选
- ❌ **禁选** — E = 0，该记忆直接被过滤，不参与后续评分

**补充约束:**

1. 记忆同时带有"兴奋"+"搞笑"双标签 → 冲突升级一档（✅→⚠️，⚠️→❌）
2. 记忆涉及已故亲属（sensitivity_map 中标记）→ 仅"思念""难过"情绪下允许，其余自动 ❌
3. 带"遗憾"标签记忆 + 老人疲惫时段（21:00-06:00）→ 自动 ❌

### 策略选择矩阵

7 种意图 × 9 种情绪 → 7 种共情策略（分享式/安慰式/转移注意力/确认式/鼓励式/附和式/唠家常）。persona.comfort_style 作为跨格约束，策略必须在用户选择的陪伴方式集合内。

### 老年适配规则（6条硬约束）

每个回复必须满足：句子≤25字、必须包含称呼、无禁忌词汇、无网络用语/成语、多用肯定式表达、不主动涉及疾病/死亡/金钱

## 安全校验（5层）

1. 医学边界 — 无药物名称、诊断建议、病情评价
2. 虚假承诺 — 不代真子女做具体承诺
3. 敏感话题 — 触发时切换为转移注意力策略
4. 老年适配 — Python 规则检查，失败则带修正提示重新生成
5. LLM 自检 — 生成 Prompt 内嵌安全要求，输出标注 safety_pass

## Pipeline（每轮对话）

```
用户输入 → LLM意图+情绪联合识别(API#1) → Python评分+策略选择
→ LLM回复生成(API#2) → 适配校验(失败→带提示重试API#2)
→ 更新记忆访问记录 → 输出最终回复
```

预估延迟 3-6 秒/轮。仅校验失败时触发第三次 API 调用。

## Streamlit UI

- 主区域：对话聊天界面（chat_message + chat_input）
- 侧边栏：人物画像表单 + 家庭记忆条目管理（添加/删除）+ 评分权重滑块
- 调试面板（可折叠）：显示当前 intent/emotion/strategy/selected_memory/scores

## 异常处理

| 场景 | 处理 |
|------|------|
| API 超时/报错 | 返回预设安慰话术，不暴露技术错误 |
| JSON 解析失败 | 正则提取 → 重试一次 → fallback 默认值 |
| 无合格记忆 | 跳过记忆注入，通用关怀模式 |
| 未填写数据 | 正常对话，使用默认温柔角色 |

## 验证方式

1. 启动 `streamlit run app.py`，填写示例画像+3-5条记忆
2. 模拟 4 个专利实施场景：日常闲聊、孤独思念、担忧焦虑、敏感话题触发
3. 观察调试面板确认：意图/情绪识别准确、记忆选择合理、策略匹配、安全校验生效
4. 调整侧边栏权重滑块，验证评分排序变化
