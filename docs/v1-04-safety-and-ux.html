04 — 安全校验 & 用户体验

5层安全 · 老年适配 · 毛玻璃 UI · 对话增强

1. 五层安全校验

1医学边界

  • 禁止药物名称/剂量
  • 禁止诊断建议
  • 禁止病情评价

2虚假承诺

  • "明天就回来看你"
  • "以后再也不走了"
  • 不代真人做承诺

3敏感话题

  • 死亡、遗产、重病
  • 切换为转移注意力策略
  • 夜间遗憾标签保护

4老年适配

  • 句长≤60字
  • 必须含称呼
  • 无网络用语/英文

5LLM 自检

  • 生成 Prompt 内嵌安全规则
  • 失败→带修正提示重试

2. 老年适配规则

✅ 必须满足

句子长度≤ 60 字
称呼使用每段回复 ≥1 次
禁忌词汇禁用死/癌症/遗产等
网络用语禁英文缩写/网络梗
肯定表达多用"是啊""对的"
回复长度2-4 句，不过于简短

🔧 违规处理

check_elderly_adaptation(response, appellation) → {pass: bool, issues: list}
失败 → 注入修正提示 → LLM 重新生成

3. 对话增强

🎯 角色自动切换

老人说"承文啊…" → LLM 提取 talk_to="刘承文" → 模糊匹配 persona → 自动切换 → 用儿子口吻回复。 未指定对象 → 通用模式（老人画像 + 策略）。

💬 话题联想展开

回复后可自然延伸话题—— "对了妈，说到槐花，上次您做的那个卜拉，承文到现在还念叨呢…" 根据记忆和老人画像联想相关内容，让对话更生动自然。

👥 多角色感知

检测提及人物 → 注入该人物画像 + 偏好档案 + 关系表 + Top-5 记忆。 如提到"小红"→ AI 知道小红是孙女、性格活泼、爱看电视剧。

🛡️ API 健壮性

10s 超时 + 自动重试 2 次 + JSON 解析容错（正则 fallback）。 异常时返回预设安慰话术，不暴露技术错误。
